# Security Policy

## Supported versions

Security fixes are provided for the latest published minor version of `@dhinesh-se/angular-components` unless maintainers document another support window.

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability. Instead, use GitHub private vulnerability reporting when available, or contact the repository owner from the GitHub organization/user profile.

Include:

- Affected package version.
- A clear description of the issue.
- Steps to reproduce or proof-of-concept details.
- Any known workarounds.

Maintainers should acknowledge valid reports, evaluate severity, prepare a fix, publish a patched version, and document the issue after users have a reasonable upgrade path.
